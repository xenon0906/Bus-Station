Bus-Station Reservation System

The "Bus-Station Reservation System" is a comprehensive and user-friendly application developed in C. This project is the culmination of my learning as a first-year computer science student, showcasing my understanding of fundamental programming concepts and the C language. This system simplifies the process of reserving bus tickets, viewing bus schedules, and managing bookings efficiently.

Features and Highlights-

1. User Authentication:
   - Secure login system to ensure authorized access to the application.
   - Username and password verification to maintain user privacy and security.

2. Bus List Viewing:
   - Users can view a list of available buses, including their names and details.
   - Clear and organized display of bus options for easy selection.

3. Ticket Booking:
   - Simple and intuitive interface for booking bus tickets.
   - Users can select the desired bus, choose the number of tickets, and enter passenger details.
   - Dynamic seat availability check and update.

4. Booking Management:
   - Ability to cancel bookings and update the seat availability accordingly.
   - Display of booked seats and passenger details for easy management.

5. Bus Status Board:
   - Real-time status board displaying the current booking status of each bus.
   - Visualization of seat occupancy and availability for quick reference.

6. File Handling:
   - Persistent storage of booking details using file handling.
   - Efficient reading and writing of seat numbers and passenger names to/from text files.

7. Error Handling:
   - Robust error handling to manage invalid inputs and ensure smooth operation.
   - Messages and prompts to guide users through the system.

8. System Messages:
   - Informative messages and prompts to guide users through each step of the booking and cancellation process.
 
Technical Highlights-

- Array Usage: Efficient use of arrays to manage and store bus names, passenger details, and seat numbers.
- String Manipulation: Proficient handling of strings for user input, file operations, and status display.
- File I/O Operations: Implementation of file input/output to save and retrieve booking data, ensuring data persistence.
- Conditional Statements: Use of conditional statements for decision-making processes, such as seat availability checks and booking confirmations.
- Loops: Utilization of loops for repetitive tasks like displaying bus status and managing multiple bookings.
- User-defined Functions: Modular code design with user-defined functions for specific tasks, enhancing code readability and maintainability.
